MCP Server quick notes

Health checks
- Server URL: http://localhost:8000
- /health should return status ok
- /chat accepts POST JSON: {"question": "...", "language": "english"}

Provider configuration
- Edit mcp_server/.env
- Set LLM_PROVIDER to one of: groq, gemini, openai, ollama
- Set matching credentials:
    - GROQ_API_KEY for groq
    - GEMINI_API_KEY for gemini
    - OPENAI_API_KEY for openai
    - OLLAMA_URL for ollama
- Set LLM_MODEL to the model you want for that provider

Data flow (PHP chat page)
User (chat.php) -> chat.js -> chat_api.php
    -> PHP LLM/tool orchestration (provider from env)
    -> Ananya API endpoints (api.php/...)
    -> final answer -> user

Data flow (optional Python MCP server)
User/client -> MCP Server (/chat)
    -> Provider selected by mcp_server/.env
    -> Tool calls -> PHP api.php endpoints
    -> final answer -> user

Start Python MCP server
cd c:\xampp\htdocs\ananya\mcp_server
.\venv\Scripts\python.exe server.py
